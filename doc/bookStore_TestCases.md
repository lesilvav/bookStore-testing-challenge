## Catalog

### TC-001: View Book Catalog

- **Description:** Verify that the public catalog page renders a paginated table with book metadata.
- **Trace to User Story:** US-001
- **Priority:** High
- **Type:** UI
- **Can be Automated:** Yes
- **Preconditions:** User is not logged in; browser is on a clean state.
- **Test Data:** N/A
- **Steps:**
  1. Navigate to `https://demoqa.com/books`.
  2. Wait for the catalog table to load.
- **Expected Result:** The table displays columns `Image`, `Title`, `Author`, and `Publisher`, with at least one book row. Pagination controls are visible.
- **Teardown:** N/A

---

### TC-002: Search Catalog with Matching Term

- **Description:** Verify that catalog search filters results by title, author, or publisher.
- **Trace to User Story:** US-002
- **Priority:** Medium
- **Type:** UI
- **Can be Automated:** Yes
- **Preconditions:** User is on `/books`.
- **Test Data:** Search term: `JavaScript`
- **Steps:**
  1. Focus the search textbox labeled "Type to search".
  2. Type `JavaScript` and submit the search.
- **Expected Result:** The table refreshes to display only books whose title, author, or publisher contains `JavaScript`.
- **Teardown:** Clear the search field to restore the full catalog.

---

### TC-003: Search Catalog with No Matching Term

- **Description:** Verify behavior when the search term does not match any book.
- **Trace to User Story:** US-002
- **Priority:** Medium
- **Type:** UI
- **Can be Automated:** Yes
- **Preconditions:** User is on `/books`.
- **Test Data:** Search term: `xyznonexistent12345`
- **Steps:**
  1. Focus the search textbox.
  2. Type `xyznonexistent12345` and submit the search.
- **Expected Result:** The table contains no rows.
- **Teardown:** Clear the search field.

---

### TC-004: View Book Details from Catalog

- **Description:** Verify that clicking a book title navigates to the book detail view.
- **Trace to User Story:** US-003
- **Priority:** High
- **Type:** UI
- **Can be Automated:** Yes
- **Preconditions:** User is on `/books`.
- **Test Data:** Book title: `Git Pocket Guide`
- **Steps:**
  1. Locate the row containing `Git Pocket Guide`.
  2. Click the book title link.
- **Expected Result:** The URL becomes `/books?search=<ISBN>` and the detail page displays ISBN, Title, Sub Title, Author, Publisher, Total Pages, and Description.
- **Teardown:** Click "Back To Book Store" or navigate back to `/books`.

## Authentication

### TC-005: Create User via API

- **Description:** Verify that a new user account can be created through the public API for test-setup purposes.
- **Trace to User Story:** US-004
- **Priority:** High
- **Type:** API
- **Can be Automated:** Yes
- **Preconditions:** N/A
- **Test Data:** Unique username and password generated by `@faker-js/faker`.
- **Steps:**
  1. Send `POST /Account/v1/User` with `{ userName: <unique>, password: <valid> }`.
- **Expected Result:** Response status is `201` (or success as documented) and the response contains the created `userID` and `username`.
- **Teardown:** Store the `userID` for cleanup; delete the user at the end of the test suite.

### TC-007: Log In with Valid Credentials

- **Description:** Verify that a registered user can log in and is redirected to `/profile`.
- **Trace to User Story:** US-005
- **Priority:** High
- **Type:** UI
- **Can be Automated:** Yes
- **Preconditions:** A test user exists (created via API in TC-005 or pre-existing fallback account `csilva`).

  1. Navigate to `/login`.

### TC-011: View Profile with Books via API-Driven Setup

- **Description:** Verify that books added through the API appear correctly in the UI profile.
- **Trace to User Story:** US-007
- **Priority:** High
- **Type:** UI + API
- **Can be Automated:** Yes
- **Preconditions:** A test user exists and has a valid Bearer token.
- **Test Data:** ISBN of a known book (e.g., `9781449325862`).
- **Steps:**
  1. Authenticate via API and obtain a Bearer token.
  2. Call `POST /BookStore/v1/Books` to add the book to the user's collection.
  3. Log in to the UI and navigate to `/profile`.
- **Expected Result:** The profile table displays the added book with its image, title, author, publisher, and a delete icon.
- **Teardown:** Call `DELETE /BookStore/v1/Books?UserId=<id>` to clear the collection; delete the test user.

### TC-013: Add Book to Collection via API

- **Description:** Verify that the API allows adding a book to a user's collection.
- **Trace to User Story:** US-008
- **Priority:** High
- **Type:** API
- **Can be Automated:** Yes
- **Preconditions:** A test user exists and has a valid Bearer token.
- **Test Data:** `{ userId: <id>, collectionOfIsbns: [{ isbn: "9781449325862" }] }`
- **Steps:**
  1. Send `POST /BookStore/v1/Books` with the payload.
  2. Send `GET /Account/v1/User/{UUID}`.
- **Expected Result:** The `books` array in the user response contains the added book.
- **Teardown:** Delete the book from the collection and delete the test user.
